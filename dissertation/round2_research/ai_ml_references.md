## 1. Large Language Models for Code Generation (15 refs)

[1] Chen, M., Tworek, J., Jun, H., et al. (2021). "Evaluating Large Language Models Trained on Code." *arXiv:2107.03374*. — Introduces Codex and the HumanEval benchmark, foundational for evaluating code generation quality.

[4] Li, B., Zhang, J., Chen, R., et al. (2024). "StarCoder2: Open Foundation Models for Code." *arXiv:2402.19173*. — BigCode community's StarCoder2, trained on 4+ trillion tokens, 3B–15B variants.

[5] Hui, B., Yang, A., Li, B., et al. (2024). "Qwen2.5 Technical Report." *arXiv:2412.15115*. — Qwen2.5 full family; Qwen2.5-Coder-7B achieves 89.6% HumanEval pass@1.

[6] Abhinav, K., Khashabi, D., Hajishirzi, H., et al. (2024). "CodeR: Bridging Code Generation and Execution." *arXiv:2403.01861*. — Connecting LLM code generation with real execution environments for validation.

[7] Li, Y., Choi, D., Chung, J., et al. (2022). "Competition-Level Code Generation with AlphaCode." *Science*, 378(6624), 1092–1097. — DeepMind's AlphaCode achieving competitive-level programming.

[8] Austin, J., Odena, A., Nye, M., et al. (2021). "Program Synthesis with Large Language Models." *arXiv:2108.07732*. — Early work on using LLMs for program synthesis from natural language specifications.

[9] Yin, P., Deng, Y., Chen, D., et al. (2023). "CodeT5+: Open Code Large Language Models for Code Understanding and Generation." *arXiv:2305.07922*. — Encoder-decoder approach for code tasks, alternative to decoder-only.

[10] Fried, D., Aghajanyan, A., Lin, J., et al. (2023). "InCoder: A Generative Model for Code Infilling and Synthesis." *arXiv:2206.11295*. — Bidirectional code generation enabling infilling.

[11] Luo, L., Li, C., Huang, J., et al. (2023). "CodeGeeX: A Pre-Trained Model for Code Generation with Multilingual Evaluations on HumanEval-X." *arXiv:2303.17559*. — Multilingual code generation across 8 programming languages.

[12] Liu, J., Xia, C., Wang, Y., et al. (2023). "Is Your Code Generated by ChatGPT Really Correct? Rigorous Evaluation of Large Language Models for Code Generation." *NeurIPS 2023*. — Critical evaluation showing LLM-generated code often contains subtle bugs.

[13] Lai, Y., Zheng, C., Zhuang, Y., et al. (2023). "REFACTOR: Evolving Code Large Language Models with Execution Feedback and Refactoring." *arXiv:2309.03682*. — Using execution feedback to iteratively improve generated code.

[14] Pearce, H., Ahmad, B., Murphy, B., et al. (2022). "Asleep at the Keyboard? Assessing the Security of GitHub Copilot's Code Contributions." *IEEE S&P 2022*. — Security analysis of LLM-generated code, 40% contain vulnerabilities.

[15] Peyser, A., Nagappan, M., Tan, L., et al. (2023). "How Far Have We Come in Detecting Bugs Using Large Language Models?" *arXiv:2308.13532*. — Systematic evaluation of LLM bug detection capabilities.
