# Human-Readable — NEXUS Project

> Plain-language summaries of the NEXUS distributed intelligence platform, written for humans who want to understand the system without reading specifications or research papers.

---

## Purpose

This directory contains human-accessible summaries of the NEXEXUS project. These documents translate the platform's technical depth — 21 specification files, a 334K-word knowledge base, 45K words of A2A language research, and five rounds of dissertation research — into clear, narrative prose. No engineering background required.

---

## Files

| File | Words | Description |
|------|------:|-------------|
| `project-overview.md` | ~7,100 | The main entry point. Covers the pitch (a fishing boat analogy), the four core problems NEXUS addresses (trust, code safety, coordination, certification), how the three-tier system works (reflex/cognitive/cloud), the A2A vision and Three Pillars, the marine reference application, eight target domains, and the current status and roadmap. |
| `simplest-system-tomorrow.md` | ~10,200 | A practical 7-day build guide for a minimum viable NEXUS using $37 of hardware. Includes a stripped-down VM (5 opcodes), a simplified trust algorithm (3 parameters), a complete bill of materials, wiring diagrams, day-by-day build steps with code samples, and an "A2A Tomorrow" variant that shows how to upgrade the MVP with agent-generated code. |
| `progression-path.md` | ~9,500 | The complete engineering roadmap from Phase 0 (current state — specifications only) through Phase 5 (post-human-code system at 36 months). Includes honest inventory of what exists and what doesn't, detailed BOMs and team requirements for each phase, success criteria, risk analysis, and budget estimates totaling $2.6M–$3M over 36 months. |

**Total: ~26,800 words.**

---

## Reading Order

1. **`project-overview.md`** — start here. Understand what NEXUS is, why it exists, and how it works.
2. **`simplest-system-tomorrow.md`** — get hands-on. Build something tangible in a week.
3. **progression-path.md`** — see the full trajectory. Understand what it takes to go from specifications to a production system.
