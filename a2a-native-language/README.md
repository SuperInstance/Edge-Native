# A2A-Native Language Research

## Agent-to-Agent Native Programming for NEXUS

This directory contains the complete research corpus for designing a programming language where **agents are the first-class interpreters** — not humans. The language enables any agent, given the right system prompt (compiler), runtime (equipment), and hardware (vessel), to actualize a user's intention directly to the capability of the underlying hardware.

### The Three Pillars

| Pillar | Role | Analogy |
|--------|------|---------|
| **System Prompt** | Compiler frontend — translates intention to bytecode | Agent's "understanding" |
| **Equipment** | Runtime environment — capability provider between bytecode and metal | OS + drivers + VM |
| **Vessel** | Hardware platform — defines capability boundaries and constraints | Physical robot |

### Key Concepts

- **Agent-Annotated Bytecode (AAB)**: 8-byte core instruction + TLV metadata trailer for agent interpretation
- **29 New Opcodes**: Intent, communication, capability negotiation, and safety augmentation categories
- **Intention Blocks**: Fundamental program unit (replaces functions/classes) — goal + constraints + verification + failure narrative
- **0.5x Trust Rule**: Agent-generated bytecode earns trust at half the rate of human-authored code
- **Backward Compatible**: All new opcodes are NOP on existing ESP32 VM — zero firmware changes required
